tablas y columnas: 

-> tb_camioneros (DNI, nombre, telefono, direccion, salario, poblacion).
-> tb_paquetes_transportados (cod_paquete, descripcion, destinatario, dir_destinatario).
-> tb_provincias (cod_provincia, nombre).
-> tb_camiones (matricula, modelo, tipo, potencia).

¿Qué tipo de relación siguen tb_camioneros y tb_paquetes_transportados?:

-> Un camionero distribuye muchos paquetes y un paquete solo puede ser distribuido por un camionero.
-> Un paquete solo puede llegar a una provincia, sin embargo a una provincia pueden llegar varios paquetes.
-> Un camionero puede conducir diferentes camiones en fechas diferentes, y un camión puede ser conducido por varios camioneros.

cod_paquete de tb_paquetes_transportados se comporta como la primary key, después, esta va a ser
la referencia que vamos a capturar desde tb_camioneros (foreign key).

cod_provincia de tb_provincias se comporta como la primary key, esta va a ser la referencia que
vamos a capturar desde tb_paquetes_transportados.

create table tb_paquete(
cod_paquete int primary key,
descrip varchar(50),
destinatario varchar(30),
dir_destino varchar(40),
dni varchar(30),
cod_prov int,
constraint fk_camionero_paquete foreign key (dni) references tb_camionero (dni),
constraint fk_prov_paquete foreign key (cod_prov) references tb_provincia (cod_prov)
);

create table tb_camionero(
dni varchar(30) primary key,
nombre varchar(30),
telefono varchar(20),
dir_camionero varchar(40),
salario varchar(30),
poblac varchar(40)
);

create table tb_provincia(
cod_prov int primary key,
nombre varchar(30)
);

create table tb_camion(
matricula varchar(30) primary key,
modelo varchar(30),
tipo varchar(20),
potencia varchar(20)
);

create table tb_conduce(
fecha date,
dni varchar(30),
matricula varchar(30),
constraint fk_camionero_conduce foreign key(dni) references tb_camionero(dni),
constraint fk_camion_conduce foreign key(matricula) references tb_camion(matricula)
);
