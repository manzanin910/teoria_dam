--- Comandos para actualizaciones e instalación de paquetes del sistema ---
sudo apt update => actualizar por primera vez.
apt list --upgradable
sudo apt install [nombre paquete]
sudo apt upgrade => para actualizar los paquetes ya instalados a la última versión disponible en los repositorios oficiales.
sudo apt remove <nombre paquete>



--- Comandos para acceso de archivos y directorios ---
cd .. (directorio padre)
cd . (directorio hijo)

cd / => cambia al directorio raiz (change directory)
cd ~ => cambia al directorio actual.

ls (linux) => enlista todos los archivos de la carpeta donde los encontremos (list files).
dir (windows) => lo mismo pero en windows, muestra los archivos y demás.

mkdir [nombre de directorio] (linux) => crear directorios (make directory).
md dir (windows) => lo mismo pero en windows, para crear directorios.



--- Directorios importantes de LINUX ---
/bin => donde se encuentran los ejecutables (binary)
/sbin => ejecutables del sistema (binary)
/boot => para el arranque del sistema.
/dev => donde se encuentran los archivos del sistema (device).
/home => espacio reservado para el usuario.
/lib => librerias necesarias para que funcionen los ejecutables.
/root => directorio del superusuario.
/tmp => ("temp" de temporal) archivos temporales.
swapfile => fichero de intercambio de memoria.



--- Directorios importantes de WINDOWS ---
\Archivos de Programa => programas instalados.
\Archivos de Programa (x86) => utilizada para almacenar programas de 32 bits (si tu sistema es de 64 bits)
\ProgramData => Es una carpeta oculta que contiene datos que necesitan las aplicaciones.
\PerfLogs => registro de rendimiento de SO.
\Usuarios
\Windows\System32 => contiene las DLL (librerías) según la versión en uso (64 o 32 bits).
\Windows\SysWOW64 => Solo sale en dispositivos en 64 bits, solo almacena DLL de 32 bits.



--- Propiedades de comandos (LISTA) (Se pueden combinar) ---
ls -l => lista en formato largo con todos los datos.
ls -lh => lista en formato largo y con el peso de cada fichero.
ls -s => lista con el tamaño en bloques (es el peso también).
ls -a => incluye los archivos ocultos (lo hace con puntos);
ls -i => incluye el numero de i-nodo (el identificador del i-nodo) de cada archivo.
ls -R => lista recursivamente el contenido.



--- Gestión de archivos por comandos en Linux ---
touch [nombre_fichero] => crear un nuevo fichero.
ln [nombre_fichero] [nombre_enlace] => para crear enlaces duros (actua como una copia) (ln => link).
ln -s [nombre_fichero] [nombre_enlace] => enlace blando o simbólico (solo actua como referencia o acceso directo).



--- Eliminación de ficheros ---
rm [lista_de_ficheros_o_directorios] => elimina los ficheros que quieras (rm => remove).
rm -i [nombre_fichero] => pide confirmación antes de borrar.
rm -f [nombre_fichero] => para forzar el borrado (por si está protegido).
rm -r [lista_directorios] => elimina directorios que no están vacíos (ojo con esto).
rmdir [lista_directorios] => elimina directorios que están vacíos.



--- Wildcards (comodines) ---
* => todos lo ficheros ; ejemplo: rm fich* => todos los ficheros que empiecen por "fich" me los borra. ; rm fich*txt => todos los ficheros que empiecen por fich y acaben por txt me los borra.
? => solo sustituye a un caracter ; ejemplo: rm fichero? => me borraría todos los ficheros que se llamen "fichero" + una letra => fichero1, fichero2.



--- Copiar archivos y directorios ---
cp [lista_ficheros_origen] [destino] => copiar ficheros a otro lugar (cp => copy).
cp -i ... => pide confirmación.
cp -r ... => copia todos los directorios, subdirectorios y ficheros a un destino.



--- Mover y renombrar archivos y directorios (la diferencia entre copiar y mover es que mover elimina el original) ---
mv [lista_ficheros_origen] [destino] => para mover archivos y directorios.
mv -i ... => pide confirmación al moverlo.
mv -u ... => cuando hay un archivo o directorio con el mismo nombre que en origen, solo lo mueve a destino si el de origen es mas reciente.



--- Mostrar contenido de archivos ---
cat [lista_archivos] => muestra los ficheros por pantalla (cat => concatenate).
more ... => muestra los ficheros página a página (extensión).
less ... => igual que more pero más potente (para ficheros grandes) (extensión de la extensión).
head [numero] ... => muestra el encabezado de cada fichero (primeras lineas => numero) (head => encabezado).
tail [numero] ... => muestra las últimas lineas de un fichero (footer).

-> Si por ejemplo queremos fusionar varios ficheros en uno => primero agrupo todos con cat:
cat fichero? (recuerda la wildcard de un solo caracter) > archivo_conjunto

">>" => añade el contenido de un archivo inicial al final del archivo destino (a no ser que no exista anteriormente).
">" => sobrescribe el contenido del archivo destino (a no ser que no exista anteriormente).



--- Escribir sobre archivos de texto en la consola ---
-> Para escribir en la consola sobre un archivo de texto, necesitamos usar un editor de texto, en este caso se usa Nano o Vim para poder escribir.
nano [nombre_archivo] => para abrir el editor de texto de nano y editar un archivo de texto.



--- Abrir un navegador y un archivo en él desde la consola ---
firefox [ruta_de_archivo_o_url] => abre firefox con las indicaciones o sin ellas.
firefox -new -window => abre en una nueva pestaña.




--- Ver metainformación de un fichero (numero de caracteres, lineas...) ---
wc [nombre_fichero] => información estadística sobre el archivo (word count)
